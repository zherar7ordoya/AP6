IController Interface  See Also  Send Feedback

The IController type exposes the following members.

Properties

  NameDescription
Task
Links controller to its context ITask object. The framework takes care of setting this property, so that every controller can access its task (see the example at the bottom).
View
Links controller to its view. The framework takes care of setting this property for every controller instance. Thus, in full accordance to the Model-View-Presenter pattern, any controller may access its view (see the example in the bottom).
